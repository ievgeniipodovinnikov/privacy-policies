Privacy Policy for QA Helper
Last updated: March 24, 2026

1. Introduction
QA Helper ("the Extension") is a Chrome extension designed to help QA engineers and developers capture bug reports with screenshots, element information, and environment details. This privacy policy explains what data the Extension collects, how it is used, and your rights regarding your data.

2. Data Collection and Storage
QA Helper does NOT collect, transmit, or store any personal data on external servers.

All data generated by the Extension is stored locally on your device using Chrome's storage.local API. This includes:

Bug reports (URLs, page titles, timestamps)

Screenshots (captured as data URLs)

Element information (XPath, CSS selectors, coordinates, text content)

Notes you add to bug reports

System information (OS, browser version, screen resolution, user agent)

Integration settings (Jira domain, email, API token; Slack webhook URL)

3. How Your Data Is Used
Local storage only: All data remains on your device. No data is sent to any external server unless you explicitly configure integrations.

Integration features: If you choose to configure Jira or Slack integrations, the Extension will send bug report data directly to those services only when you click the corresponding buttons. This data includes the bug report content (URL, title, description, screenshots) and your configured credentials (Jira API token, Slack webhook URL). These credentials are stored locally and used solely for API authentication.

Screenshot saving: When you capture a bug, a screenshot file is saved to your device's Downloads folder. This file is stored locally and is not transmitted anywhere unless you manually share it.

4. Third-Party Services
QA Helper integrates with the following third-party services only if you choose to configure them:

Jira: If you provide your Jira domain, email, and API token, the Extension can create issues in your Jira project. Data sent to Jira includes bug report details (URL, title, description, screenshot). Please refer to Atlassian's Privacy Policy for information on how they handle your data.

Slack: If you provide a Slack webhook URL, the Extension can send bug reports to your Slack channels. Data sent to Slack includes bug report details (URL, title, description). Please refer to Slack's Privacy Policy for information on how they handle your data.

The Extension does not share your data with any other third parties.

5. Data Security
All data is stored locally using Chrome's secure storage APIs.

Jira API tokens and Slack webhook URLs are stored in Chrome's storage.sync (encrypted in transit and at rest).

Screenshots and bug reports are not accessible to anyone except you.

6. Your Rights
Since all data is stored locally on your device, you have full control over it:

View: All captured bugs are visible in the Extension's history.

Delete: You can delete individual bug reports or clear the entire history using the "Clear all" button.

Remove: Uninstalling the Extension will remove all locally stored data (screenshots and bug reports remain in your Downloads folder if you saved them).

7. Changes to This Policy
We may update this privacy policy from time to time. Any changes will be posted on this page, and the "Last updated" date will be revised. Continued use of the Extension after changes constitutes acceptance of the updated policy.
